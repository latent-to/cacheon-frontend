Three formats carry a submission through Optima: the **manifest** (what you submitted), the **metadata** (what your kernel is eligible for), and the **ledger** (the commit-reveal record the validator settles over). All three are data, never executed as code at intake.

## `manifest.toml` — the bundle schema

A submission is a **bundle**: a directory with a manifest plus kernel source and optional metadata.

```
my_bundle/
  manifest.toml          # DATA: which slot(s), where the source is
  kernels/
    my_kernel.py         # the kernel source
  metadata/
    my_kernel.json       # optional eligibility
```

```toml
bundle_id   = "my-silu-v1"              # unique id, chars [0-9A-Za-z._-]
abi_version = "optima-op-abi-v0"        # must be exactly this

[[ops]]                                  # one [[ops]] block per slot you target
slot     = "activation.silu_and_mul"    # the slot id (from `optima slots`)
source   = "kernels/my_kernel.py"       # bundle-relative path to your module
entry    = "silu_and_mul"               # the function name inside it
dtypes   = ["bfloat16", "float16"]      # optional eligibility filter
metadata = "metadata/my_kernel.json"    # optional
# prepare = "prepare"                    # for (prepare, forward) slots like MoE
# setup   = "setup"                      # optional one-time engine setup (advanced)
# architectures = ["sm90", "sm100"]      # optional GPU-arch gate (sm90=H100, sm100=B200)
```

| Field | Type | Description |
| --- | --- | --- |
| `bundle_id` | string | Unique id, `[0-9A-Za-z._-]` only |
| `abi_version` | string | Must equal `optima-op-abi-v0` |
| `[[ops]]` | array of tables | One block per slot; a bundle can target several slots |
| `ops[].slot` | string | The slot id from `optima slots` |
| `ops[].source` | string | Bundle-relative path to the kernel module |
| `ops[].entry` | string | Callable name inside the module (a Python identifier) |
| `ops[].dtypes` | array | Optional dtype eligibility filter |
| `ops[].architectures` | array | Optional GPU-arch eligibility filter |
| `ops[].metadata` | string | Optional bundle-relative path to the metadata JSON |
| `ops[].prepare` | string | Optional second callable (weight-prep) for `(prepare, forward)` slots |
| `ops[].setup` | string | Optional callable run once at engine init (advanced) |

The manifest is parsed and **structurally validated** (schema, ABI version) with strict **path-safety**: every source and metadata path must be relative, must resolve inside the bundle root, and may not use `..` traversal, absolute paths, or symlink escape. Duplicate slots are rejected. Parsing imports and runs nothing; loading the kernel is a separate, later, isolated step.

### The kernel contract

The `entry` function **writes into a pre-allocated output tensor and returns `None`.** The validator owns and allocates `out`; the kernel fills it. It never returns tensors and never allocates the output; that is the anti-cheat boundary (see [Harness](/docs/evaluation/harness#the-dispatcher-the-one-place-a-miner-kernel-is-called)). `(prepare, forward)` slots (MoE) name a second callable: `prepare` runs **once** at load on the raw checkpoint weights (do weight relayout or quantization-packing here; the validator caches the result), and `entry` (forward) runs every step and receives that cached object as `prepared`.

## `metadata/*.json` — eligibility

Optional but recommended. It declares what the kernel supports; outside its claim, the validator runs the baseline instead (so claim only what you actually support).

```json
{
  "op": "activation.silu_and_mul",
  "dtypes": ["bfloat16", "float16"],
  "architectures": ["sm80", "sm86", "sm89", "sm90", "sm100"],
  "graph_safe": true,
  "max_last_dim": 65536,
  "notes": "free text"
}
```

| Field | Type | Description |
| --- | --- | --- |
| `op` | string | The slot this metadata describes |
| `dtypes` | array | Dtypes the kernel supports; outside them → baseline fallback |
| `architectures` | array | GPU arches (e.g. `sm90`=H100, `sm100`=B200); outside them → baseline |
| `graph_safe` | bool | **Required `true` for a block/collective kernel to be scored under CUDA graphs.** Without it the kernel falls back to the baseline in-graph and cannot win. Declare only if the kernel truly captures (no host syncs, no data-dependent shapes inside the graph). |
| `max_last_dim` | int | Optional size bound; beyond it → baseline fallback |
| `notes` | string | Free text |

The slot's own **correctness mode** and **KL threshold** are validator-owned (in `optima/slots.py`), not miner-declared: `allclose` for numerically-equivalent ops, `matched_ratio` / `cosine` for kernels that legitimately shift numerics (attention, FP8, MLA), and a per-slot `kl_threshold` (`5e-3` default, `3e-2` for attention). See [Scoring](/docs/evaluation/scoring#gate-b--fidelity-the-load-bearing-anti-cheat).

## The commit-reveal ledger

Submission uses **commit-reveal** so nobody can copy a bundle off-chain and front-run it. The ledger is pure-Python and persists to a JSON file so the whole mechanism is testable without a chain or a GPU. **Today this JSON ledger is a local stand-in for on-chain commitments**; in a real subnet the commitments live on-chain (subtensor commit-reveal), bundles are fetched from a content-addressed store, and `hotkey` is the miner's SS58 address. The semantics are identical.

### Bundle identity

`content_hash(bundle_dir)` is a deterministic SHA-256 over the manifest plus every source file (sorted, length-prefixed, junk excluded). Any change flips the hash. Commitments bind to it and copy detection compares it.

### Commit → reveal → settle

**Commit.** During the commit window each miner posts a commitment that hides the bundle but binds them to it:

```
commitment = H(content_hash, hotkey, salt)
```

**Reveal.** During the reveal window the miner posts `(content_hash, salt)`. A reveal is accepted only if `H(content_hash, hotkey, salt)` matches a commitment *that hotkey* posted earlier. A copier who only sees a rival's bundle at reveal time has no matching prior commitment and cannot reveal it.

**Copy detection.** Cumulative across all rounds. A different hotkey's earlier reveal of the same work demotes the later one (`original=False`), matched on any of: the exact `content_hash`; a whole-bundle reformat-invariant `fingerprint` (ignores whitespace and renames); any single `slot_fingerprints` entry (a stolen slot inside a padded bundle); or per-slot file-set **containment** via `slot_file_fingerprints` (a stolen body relocated into an imported module). The earliest commit (lowest sequence) is the original. A structural skeleton fingerprint is kept **advisory** (flags rename/constant-tweak for review, never auto-demotes).

**Settle.** The validator scores all original, passing reveals and applies king-of-the-hill (`settle`, or `settle --per-slot` for per-slot champions and an emission split). A copy ties the champion, never clears the `+2%` margin, and earns 0. Ledger state is inspectable with `optima ledger`.

### Ledger records

The on-disk ledger carries a `schema_version` and the record atoms below (unknown keys from a newer schema are dropped on read; missing optional keys fall back to defaults, so records can gain fields safely):

| Record | Key fields |
| --- | --- |
| `Commitment` | `hotkey`, `commitment`, `round_id`, `seq` (monotonic; commit order = anti-copy priority) |
| `Reveal` | `hotkey`, `content_hash`, `salt`, `round_id`, `commit_seq`, `original`, `fingerprint`, `slot_fingerprints`, `slot_file_fingerprints` |
| `Score` | `hotkey`, `content_hash`, `round_id`, `score`, `kl_mean`, `passed`, `sglang_version`, `slot` |
| `EvalRecord` | `hotkey`, `bundle_hash`, `slot`, `round_id`, `score`, `passed`, `throughput`, `mean_kl`, `gsm8k_acc`, `dq_reason` |
| `Champion` | `content_hash`, `hotkey`, `score`, `round_id`, `sglang_version` |

`EvalRecord` is keyed by `(hotkey, bundle_hash)`, so an already-scored submission is never re-run. `Score` and `Champion` carry the `sglang_version` a result was measured under, so a champion crowned on a superseded `PINNED_SGLANG` is flagged stale and re-baselined (see [Scoring](/docs/evaluation/scoring#king-of-the-hill)).
