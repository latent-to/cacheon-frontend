A directory-by-directory map of the Optima repo, so a new contributor can find
the file they need. The repo is split so the parts that do NOT need a GPU
(manifest parsing, static scanning, registry/eligibility, scoring math, the
chain I/O) run and are tested anywhere, while the GPU-only pieces (kernel
correctness on device, end-to-end sglang eval) sit behind lazy imports.

The package threat model, in one sentence (`optima/__init__.py`): with
Triton/CuteDSL the miner's kernel is *Python that runs in-process*, so
artifact-level safety is impossible and the real boundary is process + GPU-
context isolation; the code is designed to shrink the miner's host surface and
keep timing/scoring outside the miner's reach.

## The narrow waist (read these first)

Three files are the deliberately-ossified contract. Everything else is free to
churn around them.

| file | what it is | why it's the waist |
|---|---|---|
| `optima/slots.py` | the `SlotSpec` catalog + `Correctness` policy — the miner-facing submission ABI | frozen, model-agnostic; see the [slot catalog](/docs/reference/slots-table) |
| `optima/seams.py` | `SEAM_ADAPTERS` — the single source of truth for sglang chokepoint adapters | the bootstrap watch-list, the `activate()` install loop, and the `compat` canary all derive from this one table |
| `docs/SLOT_CONTRACT.md` | the four invariants every slot must satisfy | the checklist a new slot is judged against |

`seams.py` is deliberately separate from `slots.py`: `SlotSpec` is the stable
contract; a seam adapter is version-pinned glue to a specific sglang internal
that churns on every `PINNED_SGLANG` bump. Coupling them would tie the stable
contract to sglang's internals.

Everything else — `cli.py`, `dispatch.py`, `manifest.py`, the integrations, the
eval harness, the experiments — is bell-work above or below the waist.

## `optima/eval/` — end-to-end evaluation

The GPU tier: the same model under stock and candidate conditions
(identical weights/seed/sampler), bracketed by baseline launches and optionally
augmented with an untimed in-engine audit pass. ~1,800 lines.

| file | lines | what |
|---|---|---|
| `_launch.py` | 323 | shared spawn-safe, tamper-resistant engine-launch context manager (marks the driver so it never imports miner code). |
| `throughput_kl.py` | 366 | end-to-end throughput + KL fidelity eval (`evaluate`); the `EvalConfig` + `evaluate()`. |
| `capability.py` | 255 | benchmark-based capability eval (`bench`) — throughput gated by task accuracy + KL. |
| `kl.py` | 269 | KL between two runs' per-position top-k token distributions. |
| `scoring.py` | 122 | noise-robust speedup scoring for a validator that cannot lock GPU clocks (the bracketing + noise-derived bar). |
| `benchmarks.py` | 367 | capability benchmarks (GSM8K / MMLU / long_math) — real tasks that double as the eval distribution. |
| `prompts.py` | 94 | per-epoch prompt sampling (fresh subset each epoch so a kernel can't special-case fixed prompts). |

## `optima/integrations/` — sglang seam adapters

Version-pinned glue to specific sglang internals. Each file patches ONE
chokepoint. These are the lower-bell adapters — re-validated on every
`PINNED_SGLANG` bump via `optima compat`. ~780 lines.

|---|---|---|---|

| `sglang_norm.py` | 57 | wires into `RMSNorm` (universal — fires on every transformer layer) | `RMSNorm.forward_cuda` |
| `sglang_attention.py` | 59 | wires into the attention layer (backend-agnostic block seam) | `RadixAttention.forward` |
| `sglang_moe.py` | 83 | wires into fused-MoE (`forward_impl`, NOT `.forward` — piecewise-graph-safe) | `FusedMoE.forward_impl` |
| `sglang_allreduce.py` | 65 | wires into the TP all-reduce (the comms waist) | `GroupCoordinator.all_reduce` |
| `sglang_arfusion.py` | 72 | wires into the fused AR+residual+RMSNorm epilogue (module-level fn rebind) | `flashinfer_allreduce_residual_rmsnorm` |
| `sglang_msa.py` | 38 | MSA-indexer seam stub (M3-specific; NOT in the generic table) | MSA backend score kernel |

## `optima/chain/` — on-chain I/O

Mirrors the sglang seam architecture: pure helpers (no SDK, no network) carry
the weight-vector math and are unit-tested directly; thin RPC wrappers lazily
`import bittensor` and are the only code that touches the chain. ~800 lines.

| file | lines | what |
|---|---|---|
| `__init__.py` | 233 | `connect`, `fetch_metagraph`, `read_revealed_commitments`, `set_weights`, `preflight` — the chain RPC wrappers. |
| `payload.py` | 82 | the on-chain submission payload `{"v":1,"h":<hash>,"u":<url>}` — encode/decode (fail-quiet). |
| `submit.py` | 27 | miner-side submission: bind a bundle to a URL, commit via timelock commit-reveal. |
| `fetch.py` | 164 | bundle transport: `package_bundle` (tar.gz) + hostile-archive `fetch_bundle` (re-hashes after extraction). |
| `validator_loop.py` | 298 | the validator loop: commitments → fetch → evaluate → settle → weights (`run_pass` / `run_validator`). |

## `optima/patchers/` — reviewed rebuild patchers

The ONLY sanctioned patcher tier for the escape hatch. Validator-shipped and
reviewed; selected by name from a bundle's `rebuild.json`. ~330 lines.

| file | lines | what |
|---|---|---|
| `apply_dep_patch.py` | 188 | applies a bundle's declared `dep_patches` to an OVERLAY copy of the pinned dependency tree (never the shared install). |
| `build_cuda_ext.py` | 143 | compiles a bundle's declared `cuda_sources` (`.cu`/`.cuh`) and pre-loads them. |

| `codec/nvfp4.py` | 122 | NVFP4 codec + weight-layout primitives (pure torch, CPU-validatable references). |
| `moe/nvfp4_megakernel.py` | 46 | `nvfp4_moe_megakernel` — the validator-owned NVFP4 MoE base kernel (GPU path is a stub; M1.2 in the build spec). |
| `collective/fused_ar_rmsnorm.py` | 111 | fused all-reduce + residual-add + RMSNorm — the M3 decode-epilogue validated kernel, first-party portable launch layer. |
| `collective/fused_ar_rmsnorm_sm103.cu` | — | the actual SM103 CUDA kernel used in the 2026-07-02 M3 validation work. |
