---
title: CPU host setup
description: Set up the always-on CPU validator. Any VPS with Docker. Holds the wallet, scans the chain, and sets weights.
---

## Prerequisites

- Ubuntu 22.04+ VPS (any provider; 1 vCPU, 2 GB RAM is enough)
- Docker and Docker Compose installed
- A Bittensor validator wallet registered on subnet 14 (mainnet) or 470 (testnet)
- Hippius S3 bucket and credentials (shared with the GPU pod)

## Configuration

Navigate to the `cacheon/validator` directory:

```bash
cd cacheon/validator
```

Next, paste the contents from the [CPU environment template](https://github.com/latent-to/cacheon/blob/main/validator/.env.cpu.example) in the Cacheon repository into a newly created `.env` file, then update the required values with your specific configuration.

```bash
cp .env.cpu.example .env
```

Once the file is configured with your credentials, load the environment variables into your current shell session:

```bash
set -a; . .env; set +a
```

The following table shows the list of environment variables in the `.env` file, their description, and whether they are required:

| Environment variables     | Required | Description                                           |
| ------------------------- | -------- | ----------------------------------------------------- |
| `CACHEON_NETWORK`         | Yes      | `finney` (mainnet) or `test` (testnet)                |
| `CACHEON_NETUID`          | Yes      | `14` (mainnet) or `470` (testnet)                     |
| `CACHEON_WALLET_NAME`     | Yes      | Bittensor wallet name                                 |
| `CACHEON_WALLET_HOTKEY`   | Yes      | Bittensor hotkey name                                 |
| `CACHEON_POLL_INTERVAL_S` | No       | Seconds between chain scans (default `300`)           |
| `CACHEON_DRY_RUN`         | No       | Set to `1` to skip `set_weights` (useful for testing) |
| `HIPPIUS_ACCESS_KEY`      | Yes      | Hippius S3 access key                                 |
| `HIPPIUS_SECRET_KEY`      | Yes      | Hippius S3 secret key                                 |
| `CACHEON_S3_BUCKET`       | No       | S3 bucket name (default `cacheon-validator`)          |

Mount your wallet at the path expected by the container. The default is `~/.bittensor` mapped to `/root/.bittensor` inside the container. Set the `BITTENSOR_DIR` variable in the `.env` file if your wallet is located elsewhere.

## Start the run

Run the following command to begin:

The setup script creates the Python virtual environment  (or activates an existing one), installs dependencies, loads the `.env` file, tears down any running CPU validator container, and starts docker compose:

```bash
bash setup-cpu.sh
```

You can opt to run in the background. To do this:

```bash
bash setup-cpu.sh --detach
docker compose -f cpu-compose.yml logs -f
```

Expected log output on a healthy start:

```
INFO  validator.chain: Preflight OK: uid=0, stake=..., validator_permit=True
INFO  validator.cpu_validator: Tick 1: block=... challengers=0 weights_set=no
```

## Dry-run mode

Set the `CACHEON_DRY_RUN=1` variable in your `.env` file before starting to run the full chain scan and challenger selection without calling `set_weights`. This is useful for verifying chain connectivity and S3 credentials before going live.

## Verify S3 connectivity (optional)

```bash
python -m validator.sync download --state-dir /tmp/verify
ls /tmp/verify/
```

If `eval_job.json` appears, the GPU pod has already written a job to S3 and the CPU host will pick it up automatically on its next tick.
