---
title: CPU host setup
description: Set up the validator host, configure wallet and SSH, and start the validator loop.
---

The CPU host runs the validator loop: it scans the chain for new commitments, dispatches eval jobs to the GPU pod over SSH, and sets weights on-chain.

## Prerequisites

- Ubuntu 22.04+ (or any Debian-based distro)
- Python 3.10+
- SSH access to your GPU pod (see [GPU pod setup](/docs/validators/gpu-pod-setup))

## 1. Run the setup script

```bash
export GITHUB_PAT=your_github_pat_here

curl -fsSL https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup-cpu.sh | bash
```

The script installs system deps, clones the repo to `~/cacheon`, creates a venv at `~/venv-cacheon`, and installs the CPU-side Python deps (`bittensor`, `bittensor-cli`, `paramiko`).

## 2. Register on the subnet

```bash
# Mainnet
btcli subnet register --netuid 14 \
  --wallet-name <name> --wallet-hotkey <hotkey>

# Testnet
btcli subnet register --network test --netuid 460 \
  --wallet-name <name> --wallet-hotkey <hotkey>
```

## 3. Configure environment

```bash
cp ~/cacheon/validator/.env.validator.example ~/.env.validator
```

Edit `~/.env.validator`:

```bash
CACHEON_NETWORK=finney
CACHEON_NETUID=14
CACHEON_WALLET_NAME=my-validator
CACHEON_WALLET_HOTKEY=default
CACHEON_POLL_INTERVAL_S=360
CACHEON_STATE_DIR=~/cacheon/state
CACHEON_DRY_RUN=0
```

### Environment variable reference

| Variable                  | Default        | Description                                         |
| ------------------------- | -------------- | --------------------------------------------------- |
| `CACHEON_NETWORK`         | `finney`       | Bittensor network (`finney`, `test`, or `ws://...`) |
| `CACHEON_NETUID`          | `14`           | Subnet UID                                          |
| `CACHEON_WALLET_NAME`     | `default`      | Wallet name                                         |
| `CACHEON_WALLET_HOTKEY`   | `default`      | Hotkey name                                         |
| `CACHEON_POLL_INTERVAL_S` | `360`          | Seconds between idle scans                          |
| `CACHEON_STATE_DIR`       | `<repo>/state` | Path for state persistence                          |
| `CACHEON_DRY_RUN`         | `0`            | Set to `1` to skip `set_weights` and GPU eval       |

## 4. Run the validator

```bash
source ~/venv-cacheon/bin/activate
source ~/.env.validator

python ~/cacheon/scripts/remote_validator.py \
  --network finney \
  --netuid 14 \
  --wallet-name <name> \
  --wallet-hotkey <hotkey> \
  --gpu-pod-ssh-host <pod-ip-or-hostname> \
  --gpu-pod-ssh-user root
```

All SSH flags can be set via env vars:

| Flag                     | Env var                          |
| ------------------------ | -------------------------------- |
| `--gpu-pod-ssh-host`     | `CACHEON_GPU_POD_SSH_HOST`       |
| `--gpu-pod-ssh-user`     | `CACHEON_GPU_POD_SSH_USER`       |
| `--gpu-pod-ssh-port`     | `CACHEON_GPU_POD_SSH_PORT`       |
| `--gpu-pod-eval-timeout` | `CACHEON_GPU_POD_EVAL_TIMEOUT_S` |

<Callout type="info" title="Cold pod timeout">
  The first eval on a cold GPU pod can take many minutes (Docker image pull, model download, vLLM
  warmup). Raise the timeout for the first run: `--gpu-pod-eval-timeout 2400`.
</Callout>

### Running in the background

```bash
tmux new -s validator
source ~/venv-cacheon/bin/activate
source ~/.env.validator
python ~/cacheon/scripts/remote_validator.py \
  --network finney --netuid 14 \
  --wallet-name <name> --wallet-hotkey <hotkey> \
  --gpu-pod-ssh-host <pod-ip> --gpu-pod-ssh-user root
# Ctrl+B, D to detach
```
