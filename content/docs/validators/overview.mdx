<Callout type="info">
  V1 launches with a single validator run by the core team, a pragmatic starting point while the
  subnet is early-stage. This documentation exists for transparency and to lay the groundwork for
  decentralized validation in V2.
</Callout>

Running the Cacheon validator requires one GPU machine:

| Requirement | Specification                                      |
| ----------- | -------------------------------------------------- |
| **GPUs**    | 4x NVIDIA H200 (fixed NVLink topology)             |
| **Docker**  | With NVIDIA Container Toolkit                      |
| **Storage** | ~300 GB (model weights + Docker images + state)    |
| **Wallet**  | Registered on subnet 14 (mainnet) or 460 (testnet) |

The validator process runs directly on this machine. It holds the wallet keys, reads the chain, orchestrates Docker evaluations, and sets weights. No separate CPU host, no SSH tunnels.

## What the validator does

```
scan chain ──> find challengers ──> pull image ──> start container
     ^                                                    |
     |             evaluate (speed + correctness)         |
     |                                                    v
set_weights <── compare against king <── stop container, GPU cleanup
```

## What you need before starting

- A Bittensor wallet registered on subnet 14 (mainnet) or 460 (testnet)
- Docker with NVIDIA Container Toolkit on the GPU machine
- A Hugging Face token (`HF_TOKEN`) for downloading model weights
- A 4x H200 GPU machine (Targon, Lium, or your own bare metal)

## Setup

Follow the [machine setup guide](/docs/validators/machine-setup) to provision the GPU machine, install dependencies, download model weights, and start the validator.

For full architecture details, see [System overview](/docs/architecture/overview).
