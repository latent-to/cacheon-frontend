Cacheon runs **one validator process** on the operator-controlled GPU machine. It holds the wallet, scans commitments, runs Docker-backed evaluations against a pinned vLLM baseline, and sets one-hot weights to the current king.

## Hardware target

**4x (or 8x) NVIDIA H200 SXM 141GB HBM3e** with a fixed NVLink topology.

## Provisioning

Ubuntu 22.04 or 24.04. SSH in, paste:

```bash
apt-get update && apt-get install -y curl sudo
export GITHUB_PAT=ghp_...
export HF_TOKEN=... # optional but recommended
curl -fsSL -u "token:${GITHUB_PAT}" "https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup.sh" | sudo -E bash
```

That runs **[`validator/setup.sh`](https://github.com/latent-to/cacheon/blob/main/validator/setup.sh)** (Docker, NVIDIA toolkit, model weights, PG19, baseline image). Fine-grained PATs: use `-H "Authorization: Bearer ${GITHUB_PAT}"` instead of `-u "token:..."`.
Already have the repo? `cd cacheon && sudo -E bash validator/setup.sh`.
Code-only refresh: `bash validator/setup.sh --pull`.

## Configuration

Copy `validator/.env.validator.example` to `validator/.env` and fill in the required values:

| Env var                   | Purpose                                                                         |
| ------------------------- | ------------------------------------------------------------------------------- |
| `CACHEON_NETWORK`         | Bittensor network: `finney` (mainnet) or `test` (testnet)                       |
| `CACHEON_NETUID`          | Subnet UID (`14` on mainnet, `470` on testnet)                                  |
| `CACHEON_WALLET_NAME`     | Bittensor wallet name                                                           |
| `CACHEON_WALLET_HOTKEY`   | Bittensor hotkey name                                                           |
| `CACHEON_GPU_COUNT`       | Number of GPUs on the machine (e.g. `4` or `8`)                                 |
| `CACHEON_MODEL_VOLUME`    | Host path to the read-only model directory mounted at `/models` in containers   |
| `CACHEON_BASELINE_IMAGE`  | Baseline image reference (default `vllm/vllm-openai:latest`)                    |
| `CACHEON_BASELINE_DIGEST` | `sha256:...` digest from `setup.sh` or `docker inspect`; required for live eval |

Optional: **`HF_TOKEN`** for non-interactive `hf login` during setup when downloading weights.

See [Architecture: validator](/docs/architecture/validator) for the full env var reference.

## Running

The validator runs as a Docker container via Docker Compose:

```bash
cd cacheon/validator
docker compose up --build
```

Docker Compose builds the validator image, creates the dual-network topology (`cacheon-internal` for miner containers, `cacheon-egress` for chain access), mounts the Docker socket, wallet, and state directory, and starts the validator loop.

To run in dry-run mode (chain scanning without Docker eval or weight setting), set `CACHEON_DRY_RUN=1` in your `.env` file.
