---
title: Machine setup
description: Provision a 4x H200 GPU machine, install Docker with NVIDIA support, download model weights, configure the wallet, and start the validator.
---

The validator runs on a single GPU machine that handles everything: chain interaction, Docker orchestration, evaluation, and weight-setting.

## 1. Create the machine

You need **4x NVIDIA H200** (or comparable multi-GPU setup with enough VRAM for 72B BF16). We provide scripts for two providers under `scripts/gpu_setup/`.

### Targon

```bash
export TARGON_API_KEY=...
python scripts/gpu_setup/create_targon_pod.py
```

### Lium

```bash
export LIUM_API_KEY=...
python scripts/gpu_setup/create_lium_pod.py
```

<Callout type="info" title="Manual / bare metal">
  Any machine with 4x H200 (or similar multi-GPU with NVLink), Ubuntu 22.04+, and Docker with NVIDIA
  Container Toolkit will work. Fixed GPU topology is important for reproducible results.
</Callout>

## 2. Install Docker + NVIDIA Container Toolkit

If not already installed:

```bash
curl -fsSL https://get.docker.com | sh
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
  sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

Verify all 4 GPUs are visible:

```bash
docker run --rm --gpus all nvidia/cuda:12.8.0-base-ubuntu22.04 nvidia-smi
```

## 3. Download model weights

```bash
export HF_TOKEN=your_hf_token_here
huggingface-cli download Qwen/Qwen2.5-72B-Instruct \
  --local-dir /workspace/models/Qwen2.5-72B-Instruct
```

This creates the weight directory that will be mounted read-only into every container at `/models`.

## 4. Pull the vLLM baseline

```bash
docker pull vllm/vllm-openai:latest
```

## 5. Run the setup script

```bash
export GITHUB_PAT=your_github_pat_here

bash -c "$(curl -fsSL -H "Authorization: token $GITHUB_PAT" \
  https://raw.githubusercontent.com/latent-to/cacheon/main/scripts/setup.sh)"
```

This clones the repo to `/workspace/cacheon`, creates a Python venv, and installs dependencies (including `bittensor` and `bittensor-cli`).

## 6. Start the vLLM baseline

```bash
docker run -d --gpus all \
  --name cacheon-baseline \
  -p 127.0.0.1:8000:8000 \
  -v /workspace/models/Qwen2.5-72B-Instruct:/models:ro \
  vllm/vllm-openai:latest \
  --model /models \
  --host 0.0.0.0 --port 8000 \
  --dtype bfloat16 --tensor-parallel-size 4
```

Verify:

```bash
curl http://localhost:8000/health
```

## 7. Register on the subnet

```bash
# Mainnet
btcli subnet register --netuid 14 \
  --wallet-name <name> --wallet-hotkey <hotkey>

# Testnet
btcli subnet register --network test --netuid 460 \
  --wallet-name <name> --wallet-hotkey <hotkey>
```

## 8. Configure environment

```bash
cp /workspace/cacheon/validator/.env.validator.example ~/.env.validator
```

Edit `~/.env.validator`:

```bash
CACHEON_NETWORK=finney
CACHEON_NETUID=14
CACHEON_WALLET_NAME=my-validator
CACHEON_WALLET_HOTKEY=default
CACHEON_POLL_INTERVAL_S=360
CACHEON_STATE_DIR=/workspace/cacheon/state
CACHEON_DRY_RUN=0
```

### Environment variable reference

| Variable                  | Default        | Description                                         |
| ------------------------- | -------------- | --------------------------------------------------- |
| `CACHEON_NETWORK`         | `finney`       | Bittensor network (`finney`, `test`, or `ws://...`) |
| `CACHEON_NETUID`          | `14`           | Subnet UID                                          |
| `CACHEON_WALLET_NAME`     | `default`      | Wallet name                                         |
| `CACHEON_WALLET_HOTKEY`   | `default`      | Hotkey name                                         |
| `CACHEON_POLL_INTERVAL_S` | `360`          | Seconds between idle scans                          |
| `CACHEON_STATE_DIR`       | `<repo>/state` | Path for state persistence                          |
| `CACHEON_DRY_RUN`         | `0`            | Set to `1` to skip eval + weights                   |

## 9. Run the validator

```bash
source /workspace/venv/bin/activate
source ~/.env.validator

python /workspace/cacheon/scripts/validator.py \
  --network finney \
  --netuid 14 \
  --wallet-name <name> \
  --wallet-hotkey <hotkey>
```

### Running in the background

```bash
tmux new -s validator
source /workspace/venv/bin/activate
source ~/.env.validator
python /workspace/cacheon/scripts/validator.py \
  --network finney --netuid 14 \
  --wallet-name <name> --wallet-hotkey <hotkey>
# Ctrl+B, D to detach
```

## Storage notes

The persistent volume at `/workspace` survives pod restarts. Model weights, venv, Docker images, and the repo checkout all live there.
