title: Validator setup

## Prerequisites

- Ubuntu 22.04+ VPS (any provider; 1 vCPU, 2 GB RAM is enough)
- Docker and Docker Compose installed
- A Bittensor validator wallet registered on subnet 14 (mainnet) or 470 (testnet)
- Hippius S3 bucket and credentials
- Targon, Lium, and/or Shadeform API keys for GPU rental (or a pre-provisioned GPU pod)
- Sufficient balance on your GPU provider: rentals fail if billing cannot cover hourly GPU charges

## Configuration

Copy the example env file and fill in the required values:

```bash
cd cacheon/validator
cp .env.example .env
```

Then, update the required values in the .env file with your specific configuration.

### Chain and identity

The following environment variables are related to the Bittensor chain and wallet identity:

| Env var                      | Required | Description                                                                                                                       |
| ---------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `CACHEON_NETWORK`            | Yes      | `finney` (mainnet) or `test` (testnet)                                                                                            |
| `CACHEON_NETUID`             | Yes      | `14` (mainnet) or `470` (testnet)                                                                                                 |
| `CACHEON_WALLET_NAME`        | Yes      | Bittensor wallet name                                                                                                             |
| `CACHEON_WALLET_HOTKEY`      | Yes      | Bittensor hotkey name                                                                                                             |
| `CACHEON_POLL_INTERVAL_S`    | No       | Seconds between chain scans (default `600`)                                                                                       |
| `CACHEON_DRY_RUN`            | No       | Set to `1` to skip `set_weights` (useful for testing)                                                                             |
| `CACHEON_VERSION_KEY`        | No       | Mechanism version passed to `set_weights` (bump when scoring or eval rules change)                                                |
| `CACHEON_SUBMISSION_FEE_RAO` | No       | Minimum submission fee in RAO (`0` disables payment gate; default `0`. Mainnet operators typically set `100000000` for `0.1` TAO) |

<Callout type="info" title="Submission fee gate">
  When `CACHEON_SUBMISSION_FEE_RAO` is greater than `0`, the CPU validator rejects commitments
  missing valid `payment_tx` / `payment_block` fields. Miners pay via `miner/commit.py`; see [Miner
  registration](/docs/miners/registration).
</Callout>

<Callout type="info" title="Bittensor wallet directory">
  Mount your wallet at the path expected by the container. The default is `~/.bittensor` mapped to
  `/root/.bittensor` inside the container; set `BITTENSOR_DIR` in `.env` if yours is elsewhere.
</Callout>

### S3 (logs only)

Structured state uses Postgres. S3 is for log artifacts (`logs/`, `container_logs/`) synced between the GPU pod and CPU validator.

| Env var              | Required | Description                                  |
| -------------------- | -------- | -------------------------------------------- |
| `HIPPIUS_ACCESS_KEY` | Yes      | Hippius S3 access key (log upload/download)  |
| `HIPPIUS_SECRET_KEY` | Yes      | Hippius S3 secret key                        |
| `CACHEON_S3_BUCKET`  | No       | S3 bucket name (default `cacheon-validator`) |

### Postgres and monitoring API

| Env var                | Required | Description                |
| ---------------------- | -------- | -------------------------- |
| `CACHEON_DATABASE_URL` | Yes      | Postgres connection string |

See [Monitoring API](/docs/monitoring) and [Architecture](/docs/validators/architecture#observability).

### GPU orchestration

The following environment variables are related to setting up the GPU configuration:

| Env var                      | Required | Description                                                                                                                                                                                                 |
| ---------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `CACHEON_GPU_SSH`            | No       | `user@host[:port]` for a pre-provisioned GPU pod (`setup-gpu.sh` already run). When set, the validator SSHs in and runs eval; search, rent, and setup are skipped. Takes precedence over provider API keys. |
| `CACHEON_GPU_POD_PROFILE`    | No       | Remote pod profile when using `CACHEON_GPU_SSH`: `targon` (default), `lium`, or `shadeform`. Controls DinD flags and docker/sudo behavior on the remote host.                                               |
| `CACHEON_PREFERRED_PROVIDER` | No       | Restrict GPU rental to `lium`, `targon`, or `shadeform`; empty searches all configured providers. Ignored when `CACHEON_GPU_SSH` is set.                                                                    |
| `CACHEON_PREFERRED_GPU`      | No       | Pin to `H200`, `H100`, `B200`, or `B300` (case-insensitive); empty uses tier A then tier B fallback. Invalid values are ignored. Ignored when `CACHEON_GPU_SSH` is set.                                     |
| `LIUM_API_KEY`               | \*       | Lium API key (\* at least one provider key, unless `CACHEON_GPU_SSH` is set)                                                                                                                                |
| `TARGON_API_KEY`             | \*       | Targon API key (\* at least one provider key, unless `CACHEON_GPU_SSH` is set)                                                                                                                              |
| `SHADEFORM_API_KEY`          | \*       | [Shadeform](https://docs.shadeform.ai/api-reference) API key (\* at least one provider key, unless `CACHEON_GPU_SSH` is set)                                                                                |
| `CACHEON_VLLM_CACHE_DIR`     | No       | Optional on persistent GPU pods (`CACHEON_GPU_SSH`): host path for vLLM `torch.compile` cache (for example `/workspace/vllm-cache`; set empty to disable).                                                  |
| `CACHEON_MAX_HOURLY_PRICE`   | No       | Maximum hourly price in cents (default `2000`)                                                                                                                                                              |
| `HF_TOKEN`                   | No       | Hugging Face token for model download on GPU pod                                                                                                                                                            |

<Callout type="info" title="GPU pod lifetime">
  When you configure a provider API key (and leave `CACHEON_GPU_SSH` unset), the validator rents one
  GPU pod on the first eval round, runs `setup-gpu.sh` once, and reuses that pod for every later
  eval until you stop the validator container (`docker compose down`). Workload and block storage
  are deleted on shutdown. **Targon** provisions a **1 TiB** volume at `/workspace`; **Shadeform**
  creates a **500 GB** volume per rental; **Lium** attaches a named volume. First eval after rent
  runs a full cold start (model download, Docker pulls); later rounds reuse the same disk.
</Callout>

## Start

The setup script creates the Python virtual environment (or activates an existing one), installs dependencies, loads the `.env` file, tears down any running validator container, and starts docker compose:

```bash
bash setup-cpu.sh
```

To run in the background:

```bash
bash setup-cpu.sh --detach
docker compose logs -f
```

Expected log output on a healthy start:

```
INFO  validator.chain: Preflight OK: uid=0, stake=..., validator_permit=True
INFO  validator.cpu_validator: Tick 1: block=... challengers=0 weights_set=no
```

When challengers are detected:

- With `CACHEON_GPU_SSH` set, the validator SSHs into your pre-provisioned pod and runs eval.
- With a provider API key configured, the validator searches for a GPU, rents once, runs setup and eval, and keeps the pod for subsequent rounds until the validator stops.

## Dry-run mode

Set `CACHEON_DRY_RUN=1` before starting to run the full chain scan and challenger selection without calling `set_weights`. Useful for verifying chain connectivity and S3 credentials before going live.

## Verify S3 connectivity (optional)

```bash
python -m validator.sync download --state-dir /tmp/verify
ls /tmp/verify/
```

You should see `logs/` and/or `container_logs/` if prior eval runs uploaded artifacts. Structured state is not on S3.
