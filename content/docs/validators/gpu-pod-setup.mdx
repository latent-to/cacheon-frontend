<Callout type="warn">
  For automatic eval on a pod you already provisioned, set `CACHEON_GPU_SSH` on the CPU validator
  (see [Validator setup](/docs/validators/setup)). This page covers one-time pod setup and fully
  manual `docker compose` runs without SSH orchestration.
</Callout>

## Prerequisites

- A fresh Ubuntu 22.04 or 24.04 VM from Targon, Lium, or Shadeform
- At least 400 GB fast disk under `/workspace`
- Hippius S3 credentials (same bucket as the validator)
- `CACHEON_DATABASE_URL` (Postgres)

## Setup script

Run this once on a fresh machine. Optionally export your Hugging Face token using `export HF_TOKEN=...` before running the setup command to speed up model downloads.

Choose the setup flow that matches your GPU provider environment:

<Tabs items={['Targon or Lium', 'Shadeform']}>
  <Tab value="Targon or Lium">
Using Lium or Targon requires running the validator inside a nested Docker / Docker-in-Docker (DinD) configuration.  Use this command to configure the environment:
    ```bash
    export CACHEON_NESTED_DIN=1
    curl -fsSL "https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup-gpu.sh" | sudo -E DEBIAN_FRONTEND=noninteractive NEEDRESTART_MODE=a bash
    ```

  </Tab>
  <Tab value="Shadeform">
 Shadeform provides a bare VM with Docker running directly on the host system. Use this command to setup Shadeform:
    ```bash
    export CACHEON_NESTED_DIN=0
    sudo apt-get install -y curl vim git # if not already installed
    curl -fsSL "https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup-gpu.sh" | sudo -E DEBIAN_FRONTEND=noninteractive NEEDRESTART_MODE=a bash
    ```

  </Tab>
</Tabs>

When a provider API key is configured or `CACHEON_GPU_SSH` is set, the orchestrator sets `CACHEON_NESTED_DIN` from the pod profile (`CACHEON_GPU_POD_PROFILE` for manual SSH, or the rented provider) automatically.

The setup script provisions system packages, Docker, GPU configuration, dependencies, models, and verifies the environment for GPU validation.

### Code-only refresh

Run the following command after the machine is already provisioned:

```bash
bash validator/setup-gpu.sh --pull
```

Runs `git pull` and `pip install -r validator/requirements.txt` only. No OS packages, no model download.

## Configuration

Navigate to the `cacheon/validator` directory:

```bash
cd cacheon/validator
cp .env.example .env
```

Then, update the required values in the `.env` file with your specific configuration. See [Validator setup](/docs/validators/setup#configuration) for more information on the environment variables.

Once the file is configured, load the environment variables into your current shell session:

```bash
set -a; . .env; set +a
```

<Callout type="info">

- The GPU pod needs `CACHEON_DATABASE_URL` (Postgres) plus Hippius credentials for log upload: `HIPPIUS_ACCESS_KEY`, `HIPPIUS_SECRET_KEY`, `CACHEON_S3_BUCKET`.
- For manual operation you must rent an 8-GPU pod (see [GPU requirements](/docs/validators/overview#gpu-requirements)). Provider-key orchestration selects 8-GPU tier configs automatically.
- The baseline digest is automatically detected by the evaluation container.

</Callout>

## Run an evaluation

Run the following command to begin the evaluation:

```bash
docker compose -f gpu-compose.yml up --build
```

Once the evaluation completes, the container automatically exits. Each run performs a single eval from start to finish:

1. **Postgres load**: pending eval job, validator state, eval progress, and fingerprint registry
2. **Prompt generation**: seeds from `block_hash`, produces 12 eval prompts (10 scored + 2 warmup)
3. **Baseline**: starts `vllm/vllm-openai:v0.22.0`, runs eval prompts once, caches results, tears down
4. **Per miner**: run eval prompts in the miner container, then teardown
5. **Teacher-forcing batch**: one scoring vLLM boot (`v0.9.2`, 600s startup) teacher-forces scored outputs from miners that passed the text-similarity pre-filter, then teardown
6. **Container logs**: full stdout/stderr saved to `state-mainnet/container_logs/` (scoring: `baseline_scoring_{block}_{timestamp}.log`)
7. **Postgres mirror**: evaluations, leader state, and eval progress written on each update
8. **S3 log upload**: pushes `logs/` and `container_logs/` only

The validator sets weights at the end of the same tick, after GPU eval finishes and the CPU reloads structured state from Postgres.

## Local GPU eval

```bash
export CACHEON_DATABASE_URL=postgresql://...
export CACHEON_STATE_DIR=/tmp/cacheon-gpu-test
mkdir -p "$CACHEON_STATE_DIR/container_logs" "$CACHEON_STATE_DIR/logs"

export CACHEON_SKIP_S3=1
export CACHEON_MODEL_VOLUME=/workspace/models/Qwen2.5-72B-Instruct
export CACHEON_MODEL_PATH=/workspace/models/Qwen2.5-72B-Instruct
# CACHEON_BASELINE_IMAGE / CACHEON_BASELINE_DIGEST from setup-gpu output

python -m validator.gpu_eval
```

Log artifacts land under `$CACHEON_STATE_DIR` (`container_logs/`, `logs/`). Evaluations and leader state are read and written via Postgres only.
