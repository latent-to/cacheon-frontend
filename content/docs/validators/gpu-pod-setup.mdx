---
title: GPU pod setup
description: Provision a 4x H200 GPU pod, install Docker with NVIDIA support, download model weights, set up the vLLM baseline, and verify the evaluation harness.
---

The GPU pod runs the evaluation harness. It needs **4x NVIDIA H200** (or comparable multi-GPU setup with enough VRAM for 72B BF16), Docker with NVIDIA Container Toolkit, and SSH access from your CPU host.

## 1. Create the pod

We provide scripts for two providers under `scripts/gpu_setup/`. Both inject SSH keys and `.env` vars automatically.

**Targon:** Open `scripts/gpu_setup/create_targon_pod.py` and replace every `# TODO` at the top with your values.

**Lium:** Edit `# TODO` fields at the top of `scripts/gpu_setup/create_lium_pod.py`.

### Targon

```bash
export TARGON_API_KEY=...
python scripts/gpu_setup/create_targon_pod.py
```

### Lium

```bash
export LIUM_API_KEY=...
python scripts/gpu_setup/create_lium_pod.py
```

<Callout type="info" title="Manual / bare metal">
  Any machine with 4x H200 (or similar multi-GPU with NVLink), Ubuntu 22.04+, Docker with NVIDIA
  Container Toolkit, and SSH access will work. Fixed GPU topology is important for reproducible
  results.
</Callout>

## 2. Install Docker + NVIDIA Container Toolkit

If not already installed on your pod:

```bash
curl -fsSL https://get.docker.com | sh
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
  sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

Verify all 4 GPUs are visible:

```bash
docker run --rm --gpus all nvidia/cuda:12.8.0-base-ubuntu22.04 nvidia-smi
```

## 3. Download model weights

```bash
export HF_TOKEN=your_hf_token_here
huggingface-cli download Qwen/Qwen2.5-72B-Instruct \
  --local-dir /workspace/models/Qwen2.5-72B-Instruct
```

This creates the weight directory that will be mounted read-only into every container at `/models`.

## 4. Pull the vLLM baseline

The vLLM baseline runs as an always-on container:

```bash
docker pull vllm/vllm-openai:latest
```

## 5. Run the setup script

SSH into the pod and run:

```bash
export GITHUB_PAT=your_github_pat_here

bash -c "$(curl -fsSL -H "Authorization: token $GITHUB_PAT" \
  https://raw.githubusercontent.com/latent-to/cacheon/main/scripts/gpu_setup/setup.sh)"
```

This clones the repo, creates a Python venv, and installs dependencies.

## 6. Start the vLLM baseline

```bash
docker run -d --gpus all \
  --name cacheon-baseline \
  -p 127.0.0.1:8000:8000 \
  -v /workspace/models/Qwen2.5-72B-Instruct:/models:ro \
  vllm/vllm-openai:latest \
  --model /models \
  --host 0.0.0.0 --port 8000 \
  --dtype bfloat16 --tensor-parallel-size 4
```

Verify:

```bash
curl http://localhost:8000/health
```

## 7. Verify the evaluation harness

```bash
source /workspace/venv/bin/activate
cd /workspace/cacheon
python tests/e2e/e2e_pod.py --device cuda --n-prompts 2
```

## 8. Note the SSH connection details

The CPU host needs:

- **SSH host**: the pod's IP or hostname
- **SSH user**: usually `root`
- **SSH port**: usually `22`

## Storage notes

The persistent volume at `/workspace` survives pod restarts. Model weights, venv, Docker images, and the repo checkout all live there.
