## Prerequisites

- A fresh Ubuntu 22.04 or 24.04 VM from Shadeform, Lambda, or Targon (Runpod and Vast.ai are excluded: they give you a container, not a VM)
- At least 400 GB fast disk under `/workspace`
- Hippius S3 credentials (same bucket as the CPU host)
- The CPU host must have detected challengers and written `eval_job.json` to S3

## Find an available pod

```bash
python scripts/gpu_search.py --best
```

This queries Shadeform, Lambda, and Targon and returns the cheapest available instance that meets the tier requirements (see [GPU requirements](/docs/validators/overview#gpu-requirements)). It does not rent the pod automatically: use the output to provision the instance manually through the provider's UI or API, then SSH in.

## Setup script

Run the following commands once on a fresh machine:

```bash
export HF_TOKEN=...     # optional: speeds up model download
sudo apt-get install -y curl vim git # if not already installed
```

```bash
curl -fsSL "https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup.sh" | sudo -E bash
```

The setup script provisions all required system packages, Docker, GPU configuration, dependencies, models, and verifies the environment for GPU validation.

### Code-only refresh

Run the following command after the machine is already provisioned:

```bash
bash validator/setup.sh --pull
```

Runs `git pull` and `pip install -r validator/requirements.txt` only. No OS packages, no model download.

## Configuration

Navigate to the `cacheon/validator` directory:

```bash
cd cacheon/validator
```

Next, paste the contents from the [GPU environment template](https://github.com/latent-to/cacheon/blob/main/validator/.env.gpu.example) in the Cacheon repository into a newly created `.env` file, then update the required values with your specific configuration.

```bash
cp .env.gpu.example .env
```

Once the file is configured, load the environment variables into your current shell session:

```bash
set -a; . .env; set +a
```

The following table shows the list of environment variables in the `.env` file, their description, and whether they are required

| Env var                   | Required | Description                                                                |
| ------------------------- | -------- | -------------------------------------------------------------------------- |
| `CACHEON_GPU_COUNT`       | Yes      | Number of GPUs on the pod (e.g. `4` or `8`)                                |
| `CACHEON_MODEL_VOLUME`    | Yes      | Host path to model weights (e.g. `/workspace/models/Qwen2.5-72B-Instruct`) |
| `CACHEON_BASELINE_IMAGE`  | Yes      | Baseline image reference (printed by `setup.sh`)                           |
| `CACHEON_BASELINE_DIGEST` | Yes      | `sha256:...` digest (printed by `setup.sh`)                                |
| `HIPPIUS_ACCESS_KEY`      | Yes      | Hippius S3 access key                                                      |
| `HIPPIUS_SECRET_KEY`      | Yes      | Hippius S3 secret key                                                      |
| `CACHEON_S3_BUCKET`       | No       | S3 bucket name (default `cacheon-validator`)                               |

## Run an evaluation

Run the following command to begin the evaluation:

```bash
docker compose up --build
```

Once the evaluation completes, the container automatically exits. Each run performs a single eval from start to finish. Here’s what takes place:

1. **S3 download**: pulls `state.json`, `eval_job.json`, `baseline_cache/` from S3
2. **Prompt generation**: seeds from `block_hash` in the eval job, produces 10 prompts
3. **Baseline**: starts vLLM on all GPUs, single-pass eval, caches results keyed by block hash
4. **Per challenger**: pull image, start container, warmup, speed pass, correctness pass, teardown
5. **Container logs**: full stdout/stderr saved to `state/container_logs/`
6. **S3 upload**: pushes updated `state.json`, logs, and baseline cache

The CPU host detects new results on its next poll and sets weights automatically.

## Verify results (optional)

On your local machine after the eval completes:

```bash
set -a && . .env && set +a
python -m validator.sync download --state-dir /tmp/verify
cat /tmp/verify/state.json | python -m json.tool
ls /tmp/verify/container_logs/
```
