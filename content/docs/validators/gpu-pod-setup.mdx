<Callout type="warn">
  This page is for manual operation only. When `CACHEON_AUTO_RENT=1` the validator handles GPU
  rental, setup, eval, and teardown automatically. See [Validator setup](/docs/validators/setup).
</Callout>

## Prerequisites

- A fresh Ubuntu 22.04 or 24.04 VM from Targon, Lium, or Shadeform
- At least 400 GB fast disk under `/workspace`
- Hippius S3 credentials (same bucket as the validator)
- The validator must have detected challengers and written `eval_job.json` to S3

## Setup script

Run this once on a fresh machine. Optionally export your Hugging Face token using `export HF_TOKEN=...` before running the setup command to speed up model downloads.

Choose the setup flow that matches your GPU provider environment:

<Tabs items={['Targon or Lium', 'Shadeform']}>
  <Tab value="Targon or Lium">
Using Lium or Targon requires running the validator inside a nested Docker / Docker-in-Docker (DinD) configuration.  Use this command to configure the environment:
    ```bash
    export CACHEON_NESTED_DIN=1
    curl -fsSL "https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup-gpu.sh" | sudo -E DEBIAN_FRONTEND=noninteractive NEEDRESTART_MODE=a bash
    ```

  </Tab>
  <Tab value="Shadeform">
 Shadeform provides a bare VM with Docker running directly on the host system. Use this command to setup Shadeform:
    ```bash
    export CACHEON_NESTED_DIN=0
    sudo apt-get install -y curl vim git # if not already installed
    curl -fsSL "https://raw.githubusercontent.com/latent-to/cacheon/main/validator/setup-gpu.sh" | sudo -E DEBIAN_FRONTEND=noninteractive NEEDRESTART_MODE=a bash
    ```

  </Tab>
</Tabs>

When `CACHEON_AUTO_RENT=1`, the orchestrator sets the `CACHEON_NESTED_DIN` environment variable for each provider automatically.

The setup script provisions system packages, Docker, GPU configuration, dependencies, models, and verifies the environment for GPU validation.

### Code-only refresh

Run the following command after the machine is already provisioned:

```bash
bash validator/setup-gpu.sh --pull
```

Runs `git pull` and `pip install -r validator/requirements.txt` only. No OS packages, no model download.

## Configuration

Navigate to the `cacheon/validator` directory:

```bash
cd cacheon/validator
cp .env.example .env
```

Then, update the required values in the `.env` file with your specific configuration. See [Validator setup](/docs/validators/setup#configuration) for more information on the environment variables.

Once the file is configured, load the environment variables into your current shell session:

```bash
set -a; . .env; set +a
```

<Callout type="info">

- The GPU pod needs these environment variables at minimum: `HIPPIUS_ACCESS_KEY`, `HIPPIUS_SECRET_KEY`, `CACHEON_S3_BUCKET`.
- Setting `CACHEON_AUTO_RENT=1` provisions 8-GPU tier configs only; for manual operation, you must rent an 8-GPU pod (see [GPU requirements](/docs/validators/overview#gpu-requirements)).
- The baseline digest is automatically detected by the evaluation container.

</Callout>

## Run an evaluation

Run the following command to begin the evaluation:

```bash
docker compose -f gpu-compose.yml up --build
```

Once the evaluation completes, the container automatically exits. Each run performs a single eval from start to finish:

1. **S3 download**: pulls `state.json`, `eval_job.json`, `baseline_cache/` from S3
2. **Prompt generation**: seeds from `block_hash` in the eval job, produces 10 prompts
3. **Baseline**: starts vLLM on all GPUs, single-pass eval, caches results keyed by block hash
4. **Per challenger**: pull image, start container, warmup, speed pass, correctness pass, teardown
5. **Container logs**: full stdout/stderr saved to `state-mainnet/container_logs/`
6. **S3 upload**: pushes updated `state.json`, logs, and baseline cache

The validator detects new results on its next poll and sets weights automatically.

## Verify results (optional)

On your local machine after the eval completes:

```bash
set -a && . .env && set +a
python -m validator.sync download --state-dir /tmp/verify
cat /tmp/verify/state.json | python -m json.tool
ls /tmp/verify/container_logs/
```
