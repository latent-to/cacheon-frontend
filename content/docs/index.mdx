**Optima** is Bittensor Subnet 14 (SN14). It is an open, on-chain competition for **inference-throughput optimization** of LLMs.

Miners submit optimized GPU **kernels** (Triton / CuteDSL) for individual typed **slots**, a single op, a fused block, or a cross-GPU collective, inside a **fixed** model the validator controls. The validator swaps each kernel into a pinned **sglang** engine, runs the model, and scores **throughput gated by output fidelity**: per-token **KL** divergence against a stock reference run, plus task accuracy on real benchmarks. Each slot has one champion; a challenger takes the title only by beating it by a margin (**king-of-the-hill**).

## Why Optima exists

Inference is the bottleneck for serving LLMs at scale. Latency, memory, and cost decide whether a model is usable in production. Optimization techniques such as _FlashAttention_, fused kernels, MoE kernels, and tensor-parallel collectives are usually benchmarked in isolation, so objective performance comparisons are impossible.

Optima turns the problem into a live, reproducible contest: same model, same hardware, same prompts, same scoring. Miners write real GPU kernels for well-defined boundaries inside the model. The only thing that matters is making the model faster while keeping its outputs correct.

See [Why it matters](/docs/concepts/why-it-matters) for the full problem statement.

## What Optima is not

- **Not a training subnet.** The model is fixed and validator-controlled. You do not finetune weights.
- **Not a container or BYO-stack.** You do not ship a Docker image serving a whole model. You submit a kernel for one typed slot, and the validator owns everything around it.
- **Not a whole-model submission.** You implement one op (an activation, a norm, an attention or MoE block, a collective) upstream of the sampler. You never produce the final tokens, which is what makes the competition cheat-resistant.

  The mechanism is validated end-to-end on real GPUs (single H100 up to gpt-oss-120b; 4×B300 for

  live-validated on Bittensor testnet (netuid **307**, 2026-07-08) — though weights stay dry-run
  there because that subnet rejects our stake, so a real weight push needs our own subnet. Still
  open: host isolation for untrusted miners (today the process boundary is a tripwire, not a wall),
  a real DB behind the durable-JSON ledger, mainnet economics, and a public leaderboard. See the
  [Roadmap](/docs/roadmap).
