<Callout title="Roadmap" type="warn">
A hosted monitoring API and public dashboard are **planned, not built**. Today, scoring results live in the commit-reveal **ledger** and are inspected on the CLI. This page will document the hosted endpoints once they exist.
</Callout>

## Today: the ledger via the CLI

Every commit, reveal, score, and championship is recorded in the commit-reveal ledger (see [Data contracts](/docs/evaluation/data-contracts#the-commit-reveal-ledger)). It is inspectable directly:

```bash
optima ledger        # dump commitments, reveals, scores, and champions
```

The ledger is a local JSON stand-in for the on-chain commitments a production subnet would carry. It holds the audit trail: which bundles were committed and revealed, which were flagged as copies, each submission's score and KL, and the standing champion per slot (with the sglang pin it was crowned under).

## Planned: a public leaderboard

A leaderboard and dashboard over the ledger are on the roadmap. This is currently a fundraising artifact rather than shipped infrastructure, so no hosted endpoints are documented here yet. The prerequisites are chain integration (on-chain commitments and content-addressed bundle fetch) and a real datastore behind the ledger; both are tracked as production gaps.
