The `inference_engine` package is the evaluation core. It owns the policy contract, the harness, scoring, prompt sampling, and the sandbox. One installable unit; lives on both the CPU server and the GPU pod.

## Module map

| Module           | Role                                                                                                                              |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `policy.py`      | Defines the `KVCachePolicy` contract. Re-exports `CacheConfig`, `AttentionOutput`.                                                |
| `passthrough.py` | Reference implementation of `KVCachePolicy`. The baseline every miner is compared to.                                             |
| `harness.py`     | Loads the model, monkey-patches attention, runs inference, collects metrics.                                                      |
| `scoring.py`     | Compares two `RunResult`s ‚Üí `ScoreResult`.                                                                                        |
| `sandbox.py`     | Layer 1 ‚Äî static AST analysis. Rejects unsafe source before execution.                                                            |
| `runner.py`      | Layer 2 ‚Äî subprocess isolation (firejail on Linux). Runs accepted policies.                                                       |
| `prompts.py`     | `sample_prompts(block_hash, n)` ‚Äî deterministic PG19 passage sampling.                                                            |
| `__init__.py`    | Public surface: `KVCachePolicy`, `CacheConfig`, `AttentionOutput`, `PassthroughPolicy`, `Harness`, `RunResult`, `sample_prompts`. |
| `__main__.py`    | `python -m inference_engine` ‚Äî loads 7B, runs `verify()`, prints baseline metrics.                                                |

## Core data types

### `CacheConfig` ‚Äî `policy.py`

Model attention topology. Passed to `setup()` once per sequence.

```
CacheConfig
  num_layers      int          # 28 for Qwen2.5-7B
  num_heads       int          # 28
  num_kv_heads    int          # 4 (GQA)
  head_dim        int          # 128
  max_seq_len     int
  dtype           torch.dtype  # float16 in production
```

### `AttentionOutput` ‚Äî `policy.py`

Returned by `attend()` at every layer, every step.

```
AttentionOutput
  output              Tensor [batch, num_heads, q_len, head_dim]
  attention_weights   Tensor [batch, num_heads, q_len, kv_len] | None
```

### `RunResult` ‚Äî `harness.py`

Returned by `Harness.run()` after prefill + decode.

```
RunResult
  output_texts        list[str]
  output_ids          list[list[int]]
  all_logits          list[Tensor]         # [num_generated, vocab_size] per prompt
  latency_s           float
  peak_memory_bytes   int                  # torch.cuda.max_memory_allocated() (monitoring only)
  policy_memory_bytes int                  # CUDA allocator delta after full run ‚Äî memory scoring uses this
```

### `ScoreResult` ‚Äî `scoring.py`

```
ScoreResult
  kl_divergence       float                # avg per-token KL in nats
  memory_reduction    float                # clamped [-1, 1]
  latency_improvement float                # clamped [-1, 1]
  score               float                # 0.6 * memory + 0.4 * latency
  disqualified        bool
  disqualify_reason   str | None
```

### `CheckResult` ‚Äî `sandbox.py` / `runner.py`

```
CheckResult
  ok      bool
  reason  str | None          # first violation found, or None if ok
```

## Inside `harness.py`

The `harness.py` file contains three execution paths:

### `verify()` ‚Äî correctness check

Run once at harness startup:

1. Generate a reference output using the **unpatched** HuggingFace model (full sequence, per decode step).
2. Score the same token sequence through the **patched** model using `PassthroughPolicy`.
3. Assert per-token KL `< 1e-4` across all positions.

Failure here means the monkey-patch has a bug ‚Äî RoPE ordering or GQA reshape. Nothing else works until it passes.

### `run(policy, prompts)` ‚Äî production eval

1. `_patch_attention(policy)` ‚Äî replace `self_attn.forward` on all 28 layers.
2. **Prefill** ‚Äî full prompt, one forward pass.
3. **Decode loop** ‚Äî one token at a time, until EOS or `max_new_tokens`.
4. `_unpatch_attention()` ‚Äî restore original forwards.
5. Collect `RunResult` ‚Äî latency, peak VRAM (`torch.cuda.max_memory_allocated`), CUDA allocator delta (`policy_memory_bytes`), logits, output_ids.

On baseline cache misses, the pod runs a throwaway warm-up pass (one prompt, one token) before the measured baseline to absorb one-time CUDA kernel compilation and allocator warm-up costs.

### `score_policy_on_sequence(policy, prompts, reference_ids)` ‚Äî teacher-forced KL

After the autoregressive `run()`, the miner's policy is evaluated on the baseline's token sequence in a single forward pass per prompt. Both paths see identical tokens, so KL reflects only numerical policy differences. This eliminates autoregressive divergence (a single flipped greedy token cascading through the rest of the sequence).

Per layer, per step:

```
Layer:
  q, k, v = QKV projections
  q, k    = apply_rotary_pos_emb(q, k, cos, sin)
  policy.write(k, v, layer_idx, positions)
  attn    = policy.attend(q, layer_idx)
  output  = o_proj(attn.output)
```

## Inside `sandbox.py`

Single-pass AST walk. First violation found is returned.

```
parse ast
  ‚îú‚îÄ SyntaxError ‚Üí CheckResult(ok=False)
  ‚îÇ
  ‚îî‚îÄ _SafetyVisitor walks tree:
       ‚îú‚îÄ relative imports?         ‚Üí fail
       ‚îú‚îÄ imports in allowlist?     ‚Üí fail
       ‚îú‚îÄ blocked bare calls?       ‚Üí fail
       ‚îú‚îÄ blocked attribute targets?‚Üí fail
       ‚îî‚îÄ _check_structure():
            - exactly one KVCachePolicy subclass
            - setup, write, attend all present
       ‚Üí CheckResult(ok=True)
```

Details on the allowlist and what does/doesn't match: see [Sandbox](/docs/architecture/sandbox).

## Inside `prompts.py`

`sample_prompts(block_hash, n, *, max_chars, min_chars)`:

```
block_hash (hex)
    ‚îÇ  sha256 ‚Üí 8 bytes ‚Üí int seed
    ‚ñº
random.Random(seed)
    ‚îÇ  shuffle PG19 row indices
    ‚ñº
for each index:
    passage = PG19_ROWS[index]
    if len(passage) < min_chars: skip
    passage = truncate_at_boundary(passage, max_chars)
    yield passage
    stop after n
```

Same block hash always produces the same passage list. Auditable ‚Äî anyone with the hash can replay the sample.

Defaults:

| Parameter   | Value      | Why                                                      |
| ----------- | ---------- | -------------------------------------------------------- |
| `max_chars` | 131,072    | ~32K tokens; KV cache reaches ~1.8 GB (~12% of peak)     |
| `min_chars` | 1,000      | Filters out PG19 frontmatter, chapter headers, OCR cruft |
| `revision`  | pinned SHA | Row ordering stable across dataset updates               |

## Public surface

Everything a caller (validator loop, pod script, or local test) needs is re-exported from `inference_engine/__init__.py`:

```python
from inference_engine import (
    KVCachePolicy,
    CacheConfig,
    AttentionOutput,
    PassthroughPolicy,
    Harness,
    RunResult,
    sample_prompts,
)
from inference_engine.scoring import score
```

Miners only need to import `KVCachePolicy`, `CacheConfig`, `AttentionOutput` for their submission ‚Äî the rest is for running the harness.

See the [Policy interface](/docs/miners/policy-interface) for what miners need to implement, and [Scoring](/docs/evaluation/scoring) for what `ScoreResult` means in practice.

Validator-side and cross-boundary types (`CommitmentRecord`, `EvaluationJob`, `EvaluationResult`, persisted `ValidatorState`) live in [Data contracts](/docs/architecture/data-contracts).
