The `inference_engine` package is the evaluation core. It owns the policy contract, the harness, scoring, prompt sampling, and the sandbox. One installable unit; lives on both the CPU server and the GPU pod.

## Module map

| Module             | Role                                                                                  |
| ------------------ | ------------------------------------------------------------------------------------- |
| `policy.py`        | Defines the `KVCachePolicy` contract. Re-exports `CacheConfig`, `AttentionOutput`.    |
| `passthrough.py`   | Reference implementation of `KVCachePolicy`. The baseline every miner is compared to. |
| `harness.py`       | Loads the model, monkey-patches attention, runs inference, collects metrics.          |
| `scoring.py`       | Compares two `RunResult`s → `ScoreResult`.                                            |
| `sandbox.py`       | Layer 1 — static AST analysis. Rejects unsafe source before execution.                |
| `runner.py`        | Layer 2 — subprocess isolation (firejail on Linux). Runs accepted policies.           |
| `prompts.py`       | `sample_prompts(block_hash, n)` — deterministic PG19 passage sampling.                |
| `__init__.py`      | Public surface: `KVCachePolicy`, `CacheConfig`, `AttentionOutput`, `PassthroughPolicy`, `Harness`, `RunResult`, `sample_prompts`. |
| `__main__.py`      | `python -m inference_engine` — loads 7B, runs `verify()`, prints baseline metrics.   |

## Core data types

### `CacheConfig` — `policy.py`

Model attention topology. Passed to `setup()` once per sequence.

```
CacheConfig
  num_layers      int          # 28 for Qwen2.5-7B
  num_heads       int          # 28
  num_kv_heads    int          # 4 (GQA)
  head_dim        int          # 128
  max_seq_len     int
  dtype           torch.dtype  # float16 in production
```

### `AttentionOutput` — `policy.py`

Returned by `attend()` at every layer, every step.

```
AttentionOutput
  output              Tensor [batch, num_heads, q_len, head_dim]
  attention_weights   Tensor [batch, num_heads, q_len, kv_len] | None
```

### `RunResult` — `harness.py`

Returned by `Harness.run()` after prefill + decode.

```
RunResult
  output_texts        list[str]
  output_ids          list[list[int]]
  all_logits          list[Tensor]         # [num_generated, vocab_size] per prompt
  latency_s           float
  peak_memory_bytes   int                  # torch.cuda.max_memory_allocated()
  policy_memory_bytes int                  # policy.memory_bytes() (self-reported)
```

### `ScoreResult` — `scoring.py`

```
ScoreResult
  kl_divergence       float                # avg per-token KL in nats
  memory_reduction    float                # clamped [-1, 1]
  latency_improvement float                # clamped [-1, 1]
  score               float                # 0.6 * memory + 0.4 * latency
  disqualified        bool
  disqualify_reason   str | None
```

### `CheckResult` — `sandbox.py` / `runner.py`

```
CheckResult
  ok      bool
  reason  str | None          # first violation found, or None if ok
```

## Evaluation flow

For a single challenger, end to end:

```
source (str, fetched from GitHub/HF)
    │
    ▼
sandbox.check(source)     ←  Layer 1 (CPU, ms)
    │
    ├─ ok=False ─▶  CheckResult returned, submission scored 0
    │
    ▼
runner.run_check(source)  ←  Layer 2 (CPU, subprocess with firejail)
    │
    ├─ worker.py writes result.json
    │      → _validate_output(shape, NaN, range, attn sum)
    │
    ├─ ok=False ─▶  CheckResult returned, submission scored 0
    │
    ▼
[SSH to GPU pod]
    │
    ▼
harness.run(PassthroughPolicy, prompts)    →  RunResult (baseline)
harness.run(MinerPolicy,        prompts)   →  RunResult (miner)
    │
    ▼
scoring.score(baseline, miner)   →   ScoreResult
    │
    ▼
[SFTP back to CPU]
    │
    ▼
validator.loop updates state, sets weights
```

## Inside `harness.py`

Two execution paths:

### `verify()` — correctness check

Run once at harness startup:

1. Generate a reference output using the **unpatched** HuggingFace model (full sequence, per decode step).
2. Score the same token sequence through the **patched** model using `PassthroughPolicy`.
3. Assert per-token KL `< 1e-4` across all positions.

Failure here means the monkey-patch has a bug — RoPE ordering or GQA reshape. Nothing else works until it passes.

### `run(policy, prompts)` — production eval

1. `_patch_attention(policy)` — replace `self_attn.forward` on all 28 layers.
2. **Prefill** — full prompt, one forward pass.
3. **Decode loop** — one token at a time, until EOS or `max_new_tokens`.
4. `_unpatch_attention()` — restore original forwards.
5. Collect `RunResult` — latency (`time.monotonic`), peak memory (`torch.cuda.max_memory_allocated`), logits, output_ids.

Per layer, per step:

```
Layer:
  q, k, v = QKV projections
  q, k    = apply_rotary_pos_emb(q, k, cos, sin)
  policy.write(k, v, layer_idx, positions)
  attn    = policy.attend(q, layer_idx)
  output  = o_proj(attn.output)
```

## Inside `sandbox.py`

Single-pass AST walk. First violation found is returned.

```
parse ast
  ├─ SyntaxError → CheckResult(ok=False)
  │
  └─ _SafetyVisitor walks tree:
       ├─ relative imports?         → fail
       ├─ imports in allowlist?     → fail
       ├─ blocked bare calls?       → fail
       ├─ blocked attribute targets?→ fail
       └─ _check_structure():
            - exactly one KVCachePolicy subclass
            - setup, write, attend, memory_bytes all present
       → CheckResult(ok=True)
```

Details on the allowlist and what does/doesn't match: see [Sandbox](/docs/architecture/sandbox).

## Inside `runner.py`

```
sandbox.check(source)
    │
    ├─ not ok ─▶  return
    │
    ▼
mkdtemp()  →  /tmp/cacheon_<uuid>/
write policy_submission.py
write worker.py
    │
    ▼
firejail on PATH?
    │
    ├─ yes ─▶ firejail --net=none --private=<workdir> ... python -u worker.py
    ├─ no  ─▶ python -u worker.py   (warning logged)
    │
    ▼
subprocess.run(timeout=300, start_new_session=True)
    │
    ├─ TimeoutExpired   ─▶ CheckResult(ok=False, reason="timeout")
    ├─ returncode != 0  ─▶ CheckResult(ok=False, reason=stderr)
    │
    ▼
read result.json
_validate_output(shape, dtype, NaN/Inf, value range, attn sum)
    │
    ├─ invalid ─▶ CheckResult(ok=False)
    └─ valid   ─▶ CheckResult(ok=True)

shutil.rmtree(workdir)     # always, even on failure
```

## Inside `prompts.py`

`sample_prompts(block_hash, n, *, max_chars, min_chars)`:

```
block_hash (hex)
    │  sha256 → 8 bytes → int seed
    ▼
random.Random(seed)
    │  shuffle PG19 row indices
    ▼
for each index:
    passage = PG19_ROWS[index]
    if len(passage) < min_chars: skip
    passage = truncate_at_boundary(passage, max_chars)
    yield passage
    stop after n
```

Same block hash always produces the same passage list. Auditable — anyone with the hash can replay the sample.

Defaults:

| Parameter   | Value        | Why                                                       |
| ----------- | ------------ | --------------------------------------------------------- |
| `max_chars` | 131,072      | ~32K tokens; KV cache reaches ~1.8 GB (~12% of peak)      |
| `min_chars` | 1,000        | Filters out PG19 frontmatter, chapter headers, OCR cruft  |
| `revision`  | pinned SHA   | Row ordering stable across dataset updates                |

## Public surface

Everything a caller (validator loop, pod script, or local test) needs is re-exported from `inference_engine/__init__.py`:

```python
from inference_engine import (
    KVCachePolicy,
    CacheConfig,
    AttentionOutput,
    PassthroughPolicy,
    Harness,
    RunResult,
    sample_prompts,
)
from inference_engine.scoring import score
```

Miners only need to import `KVCachePolicy`, `CacheConfig`, `AttentionOutput` for their submission — the rest is for running the harness.

See the [Policy interface](/docs/miners/policy-interface) for what miners need to implement, and [Scoring](/docs/evaluation/scoring) for what `ScoreResult` means in practice.

Validator-side and cross-boundary types (`CommitmentRecord`, `EvaluationJob`, `EvaluationResult`, persisted `ValidatorState`) live in [Data contracts](/docs/architecture/data-contracts).
