title: Data contracts
description: Wire formats for on-chain commitments and internal evaluation data. JSON schemas for EvaluationJob and EvaluationResult.
---

## On-chain commitment

Miners post a JSON commitment via `subtensor.set_reveal_commitment()`:

```json
{
  "image": "docker.io/miner/server:v1",
  "digest": "sha256:a3b4c5d6e7f8..."
}
```

| Field    | Type   | Description                                     |
| -------- | ------ | ----------------------------------------------- |
| `image`  | string | Full Docker image reference (registry/repo:tag) |
| `digest` | string | Image manifest digest (sha256:...)              |

The digest pins the exact image content. Tag mutations do not affect evaluation.

## EvaluationJob

Internal data structure representing one evaluation tick:

```json
{
  "schema_version": 1,
  "job_id": "uuid",
  "challenger_hotkey": "5F...",
  "image": "docker.io/miner/server:v1",
  "digest": "sha256:a3b4c5d6e7f8...",
  "prompts": [{ "role": "user", "content": "<instruction + passage>" }, "..."],
  "block_hash": "0xabc...",
  "max_tokens": 256,
  "n_warmup": 2,
  "per_prompt_timeout_s": 120,
  "baseline_image": "vllm/vllm-openai:v0.x.y",
  "model_volume": "/workspace/models/Qwen2.5-72B-Instruct"
}
```

## EvaluationResult

Result of one challenger evaluation:

```json
{
  "schema_version": 1,
  "job_id": "uuid",
  "challenger_hotkey": "5F...",
  "baseline_ttft_median_s": 1.42,
  "baseline_throughput_median_tps": 52.3,
  "miner_ttft_median_s": 1.08,
  "miner_throughput_median_tps": 67.8,
  "ttft_improvement": 0.239,
  "throughput_improvement": 0.296,
  "token_match_rate": 0.994,
  "correctness_pass": true,
  "score": 0.268,
  "disqualified": false,
  "disqualify_reason": null,
  "n_prompts": 10,
  "n_warmup": 2,
  "aggregation": "median",
  "eval_duration_s": 342.5,
  "image_digest": "sha256:a3b4c5d6e7f8..."
}
```

## Key fields

| Field                            | Type   | Description                                               |
| -------------------------------- | ------ | --------------------------------------------------------- |
| `baseline_ttft_median_s`         | float  | Median baseline TTFT across prompts                       |
| `baseline_throughput_median_tps` | float  | Median baseline throughput across prompts                 |
| `miner_ttft_median_s`            | float  | Median miner TTFT across prompts                          |
| `miner_throughput_median_tps`    | float  | Median miner throughput across prompts                    |
| `ttft_improvement`               | float  | `max(0, (baseline - miner) / baseline)` on medians        |
| `throughput_improvement`         | float  | `max(0, (miner - baseline) / baseline)` on medians        |
| `token_match_rate`               | float  | Fraction of output tokens matching baseline (all prompts) |
| `correctness_pass`               | bool   | Whether the correctness gate was passed (>= 99%)          |
| `score`                          | float  | `0.5 * ttft + 0.5 * throughput` if correct, else 0        |
| `disqualified`                   | bool   | Whether the submission was DQ'd                           |
| `aggregation`                    | string | Always `"median"` in V1                                   |

## Schema versioning

Both job and result payloads include a `schema_version` field (V1 starts at version 1). This allows rolling upgrades without breaking compatibility with persisted state files.
