## Training is solved. Serving is the bottleneck.

Most public attention still goes to training. In production, the constraints that determine whether a model is usable are different:

- **Latency** -- time-to-first-token and tokens/sec. Users and agents feel the delay.
- **Throughput** -- how many concurrent requests you can serve on fixed hardware.
- **Cost** -- GPU utilization and serving efficiency. Dollars per million tokens.

These are **inference-time** problems. Cacheon treats inference optimization as a competition and asks the network to find the best serving stack in the open.

## The optimization surface

Modern inference has dozens of knobs, and each one interacts with the others:

| Layer          | Techniques                                                  | Impact                             |
| -------------- | ----------------------------------------------------------- | ---------------------------------- |
| **Memory**     | KV cache quantization, eviction, paging, offloading         | Max context length, max batch size |
| **Compute**    | FlashAttention, custom CUDA kernels, fused operations       | Tokens/sec, prefill speed          |
| **Scheduling** | Continuous batching, chunked prefill, preemptive scheduling | Throughput under load              |
| **Decoding**   | Speculative decoding, Medusa heads, lookahead               | Output tokens/sec                  |
| **Runtime**    | Rust/C++ servers, kernel-level optimizations, memory layout | End-to-end latency                 |

Existing solutions like vLLM, TensorRT-LLM, and SGLang each make different tradeoffs across these axes. No single team has tried every combination. Cacheon creates an incentive to search this space exhaustively.

## The benchmark problem

There is no shared, verifiable benchmark for full-stack inference performance:

- vLLM publishes its own benchmarks against its own baselines
- Academic papers measure individual techniques in isolation
- Inference providers report numbers against undisclosed configurations

The **numbers don't compare.** Teams shipping production inference have no way to know which combination of techniques actually wins for their specific model and workload.

<Callout title="What Cacheon does">
  Fix the model, fix the hardware, fix the prompts, fix the scoring, and let anyone submit a full
  serving stack. The subnet produces an ordered ranking that is apples-to-apples comparable.
</Callout>

## What winning looks like

The endpoint is not a paper or a blog post. The endpoint is an inference server ranked #1 on the leaderboard, serving a top open-source model faster than vLLM on identical hardware. That is a product: the fastest way to run that model, discovered through open competition.
