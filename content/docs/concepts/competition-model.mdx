Optima runs a **per-slot king-of-the-hill** competition. For each typed slot there is at most one **champion**: the kernel with the best crownable speedup on record. A challenger takes the title only by beating the champion by a fixed margin. Emission is split across the slots that have a champion, so a miner can earn by specializing in a single slot rather than winning everywhere.

## How a round works

A round is `commit → reveal → evaluate → settle`.

1. **Commit.** Each miner posts `H(content_hash, hotkey, salt)`. This hides the bundle (nobody can tell what you committed to) but binds you to it.
2. **Reveal.** The miner posts `(content_hash, salt)`. A reveal is accepted only if `H(content_hash, hotkey, salt)` matches a commitment **that same hotkey** posted earlier.
3. **Evaluate.** The validator swaps the revealed kernel into the fixed model and scores it: a two-launch throughput measurement gated by fidelity (KL divergence plus benchmark accuracy). See [Scoring](/docs/validators/scoring).
4. **Settle.** `optima settle` takes the round's passing, original scores and applies the king-of-the-hill rule per slot.

## King-of-the-hill

Settlement is not a plain `>`. A challenger takes a slot's title only if:

```
challenger_score ≥ champion_score × (1 + margin)
```

The margin defaults to **2%**. The champion holds the emission weight for that slot until a challenger clears the margin.

Why a margin, not a strict comparison:

- Running the same kernel twice produces small score variance from GPU clock drift, scheduling jitter, and measurement noise. Without a margin, rankings could flip on noise. The scoring path also derives its bar from the *measured* baseline noise, so a genuine but tiny edge on a noisy box does not crown. See [Scoring](/docs/validators/scoring).

## Per-slot emission

`optima settle --per-slot` runs a **champion per slot** and splits emission across the slots that have one. This rewards specialists: a miner who owns a single slot is paid, rather than winner-take-all giving everything to the single best end-to-end bundle. A slot with no passing kernel simply pays no one.

## Anti-copy: commit-reveal

Submissions are evaluated in the open, so a lazy miner could try to copy the current champion and resubmit. Commit-reveal defeats this at the protocol level:

- A copier who only sees a rival's bundle at reveal time has **no matching prior commitment** for it, so they cannot reveal it. Copying at reveal time is impossible.
- If two miners independently committed to the same content, the **earliest commit is the original**; later identical ones are marked copies.

In production this is Bittensor's native commit-reveal; Optima keeps the off-chain scoring half.

## Copy detection

Bundle identity is a deterministic content hash over the manifest and every source file. Copy detection is **cumulative across rounds** (a copy in a later round is caught, not just a same-round duplicate) and fires on either:

- the **exact content hash**, or
- a **reformat-invariant fingerprint** (AST-normalized), so a reflowed, recommented, or renamed-whitespace copy with a fresh hash is still demoted.

Fingerprints are computed per slot over each op's bundle-local import closure, and the ledger compares per-file fingerprint sets by containment, so padding the bundle with an extra op, or relocating a stolen kernel body into an imported module behind a re-export shim, still demotes. Two miners merely vendoring the same public utility do not match. A structural fingerprint (names and constants blanked) additionally flags rename-plus-constant-tweak near-copies as an **advisory** for review, never an automatic demotion.

A copy **ties** the champion. It never clears the margin, so it earns nothing. The only way to earn is to genuinely beat the best kernel on record for a slot.
