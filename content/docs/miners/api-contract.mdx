## Required endpoints

### `GET /health`

Returns `HTTP 200` when the server is ready to serve requests. The validator polls this endpoint after starting your container. If it does not return `200` within the 12.5-minute startup timeout window, the container is terminated and the submission is disqualified.

```json
{ "status": "ok" }
```

### `POST /v1/chat/completions`

The validator sends eval prompts here during evaluation (`max_tokens` is **512**, ~6k input context).

**Request format:**

```json
{
  "model": "Qwen2.5-72B-Instruct",
  "messages": [{ "role": "user", "content": "<instruction with ~6k passage>" }],
  "max_tokens": 512,
  "temperature": 0,
  "stream": true,
  "logprobs": true,
  "top_logprobs": 5
}
```

**Required capabilities:**

| Parameter      | Requirement                                                                                                  |
| -------------- | ------------------------------------------------------------------------------------------------------------ |
| `temperature`  | Must support `0` (greedy decoding)                                                                           |
| `stream`       | Must support `true` (SSE streaming). Production scoring only uses `stream=true`; `stream=false` is optional. |
| `logprobs`     | Must return top-N log probabilities per output token when requested                                          |
| `top_logprobs` | Must support values up to 5                                                                                  |
| `max_tokens`   | Must respect the requested generation length                                                                 |

**Response format (streaming):**

Standard _Server-Sent Event_ (_SSE_) with `data: {"choices": [{"delta": {"content": "..."}, "logprobs": {...}}]}` chunks. Final chunk is `data: [DONE]`.

## One-pass evaluation

The validator runs **12 eval prompts** (2 warmup discarded, 10 scored) in one container session, then teacher-forces your **same outputs** on a separate scoring vLLM:

- **Speed:** ~6k context, `max_tokens=512`, streaming with logprobs. Measures end-to-end wall time vs the cached baseline (request to final token).
- **Correctness:** teacher-forcing on those same 10 outputs via `prompt_logprobs` on the scoring vLLM, not from your streamed logprobs.

The validator measures:

- **Speed:** total response time vs baseline (`speed_improvement` on the evaluation record)
- **Correctness:** teacher-forcing on the same outputs used for speed. See [Scoring](/docs/evaluation/scoring).

## Model identity

Weights are mounted read-only at `/models`. Serve them as-is in BF16.
