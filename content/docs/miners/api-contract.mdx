## Required endpoints

### `GET /health`

Returns `HTTP 200` when the server is ready to serve requests. The validator polls this endpoint after starting your container. If it does not return `200` within the 10-minute startup window, the container is terminated and the submission is disqualified.

```json
{ "status": "ok" }
```

### `POST /v1/chat/completions`

Standard OpenAI chat completions endpoint. The validator sends all prompts here during both the speed pass and the correctness pass.

**Request format:**

```json
{
  "model": "Qwen2.5-72B-Instruct",
  "messages": [{ "role": "user", "content": "<instruction with long-context passage>" }],
  "max_tokens": 256,
  "temperature": 0,
  "stream": true,
  "logprobs": true,
  "top_logprobs": 5
}
```

**Required capabilities:**

| Parameter      | Requirement                                                         |
| -------------- | ------------------------------------------------------------------- |
| `temperature`  | Must support `0` (greedy decoding)                                  |
| `stream`       | Must support `true` (SSE streaming) and `false`                     |
| `logprobs`     | Must return top-N log probabilities per output token when requested |
| `top_logprobs` | Must support values up to 5                                         |
| `max_tokens`   | Must respect the requested generation length                        |

**Response format (streaming):**

Standard _Server-Sent Event_ (_SSE_) with `data: {"choices": [{"delta": {"content": "..."}, "logprobs": {...}}]}` chunks. Final chunk is `data: [DONE]`.

**Response format (non-streaming):**

```json
{
  "choices": [
    {
      "message": { "role": "assistant", "content": "..." },
      "logprobs": {
        "content": [
          {
            "token": "token1",
            "logprob": -0.12,
            "top_logprobs": [
              { "token": "token1", "logprob": -0.12 },
              { "token": "token2", "logprob": -2.3 }
            ]
          }
        ]
      }
    }
  ]
}
```

## Two-pass evaluation

The validator evaluates your server in two passes:

### Pass 1: Speed (streaming, no logprobs)

```json
{
  "model": "Qwen2.5-72B-Instruct",
  "messages": [{ "role": "user", "content": "..." }],
  "max_tokens": 256,
  "temperature": 0,
  "stream": true
}
```

The validator measures:

- **TTFT**: wall-clock time from request send to first SSE data chunk
- **Throughput**: output tokens per second, measured from SSE timestamps

### Pass 2: Correctness (non-streaming, logprobs requested)

```json
{
  "model": "Qwen2.5-72B-Instruct",
  "messages": [{ "role": "user", "content": "..." }],
  "max_tokens": 256,
  "temperature": 0,
  "stream": false,
  "logprobs": true,
  "top_logprobs": 5
}
```

The validator compares your output against the baseline using **per-prompt first-divergence analysis** with logprob verification. See [Scoring](/docs/evaluation/scoring) for the full specification.

This pass is not timed. Take as long as you need.

## Model identity

Weights are mounted read-only at `/models`. Serve them as-is in BF16.
