## Overview

Registration is two steps:

1. Push your Docker image to a public registry
2. Commit the image reference on-chain using your Bittensor wallet

The validator scans for new commitments approximately every 5 minutes. When it finds yours, it pulls your image and evaluates it.

## Prerequisites

- [Install the Bittensor SDK](https://docs.learnbittensor.org/getting-started/installation). This is required in the environment used to run the [Cacheon miner commit script](#step-2-commit-on-chain).
- A Bittensor wallet registered on Bittensor mainnet subnet 14. For more information, see [Wallets in Bittensor](https://docs.bittensor.com/getting-started/wallets) and [Miner registration](https://docs.learnbittensor.org/miners#miner-registration).

## Step 1: Push your image

Tag and push your image to a public registry:

```bash
docker tag my-server:latest docker.io/myuser/cacheon-miner:v1
docker push docker.io/myuser/cacheon-miner:v1
```

Note the image digest (the `sha256:...` printed after push). You will need this.

To get the digest after pushing:

```bash
docker inspect --format='{{index .RepoDigests 0}}' docker.io/myuser/cacheon-miner:v1
```

## Step 2: Commit on-chain

Use the Cacheon commit script found in the [Cacheon GitHub repository](https://github.com/latent-to/cacheon/blob/main/miner/commit.py):

```bash
python miner/commit.py \
  --wallet-name <your-wallet> \
  --wallet-hotkey <your-hotkey> \
  --image "docker.io/myuser/cacheon-miner:v1" \
  --digest "sha256:abc123..." \
  --network finney \
  --netuid 14
```

This script calls `subtensor.set_reveal_commitment()` with a JSON payload containing your Docker image reference and image digest:

```json
{
  "image": "docker.io/myuser/cacheon-miner:v1",
  "digest": "sha256:abc123..."
}
```

<Callout type="warn" title="One shot per hotkey">
  Each hotkey can only commit once. If you need to submit a new server, you must register a new
  hotkey on the subnet. Test locally before committing.
</Callout>

## What happens after

The validator pulls your image and runs the [evaluation harness](/docs/evaluation/harness). If your score beats the current winner, you take the top spot.
