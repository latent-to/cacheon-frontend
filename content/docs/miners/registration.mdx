## Overview

Registration is two steps:

1. Push your Docker image to a public registry
2. Commit the image reference on-chain using your Bittensor wallet

The validator scans for new commitments every epoch (~360s). When it finds yours, it pulls your image and evaluates it.

## Step 1: Push your image

Tag and push your image to a public registry:

```bash
docker tag my-server:latest docker.io/myuser/cacheon-miner:v1
docker push docker.io/myuser/cacheon-miner:v1
```

Note the image digest (the `sha256:...` printed after push). You will need this.

To get the digest after pushing:

```bash
docker inspect --format='{{index .RepoDigests 0}}' docker.io/myuser/cacheon-miner:v1
```

## Step 2: Commit on-chain

Use the Cacheon commit script:

```bash
python miner/commit.py \
  --wallet-name <your-wallet> \
  --wallet-hotkey <your-hotkey> \
  --image "docker.io/myuser/cacheon-miner:v1" \
  --digest "sha256:abc123..." \
  --network finney \
  --netuid 14
```

This calls `subtensor.set_reveal_commitment()` with a JSON payload:

```json
{
  "image": "docker.io/myuser/cacheon-miner:v1",
  "digest": "sha256:abc123..."
}
```

<Callout type="warn" title="One shot per hotkey">
  Each hotkey can only commit once. If you need to submit a new server, register a new hotkey. Test
  locally before committing.
</Callout>

## What happens after

1. Validator detects your commitment in the next epoch scan
2. Validator pulls your Docker image by digest
3. Container starts with GPU access, validator polls `/health`
4. Prompts are sent, speed and correctness are measured
5. Score is computed and compared against the king
6. If you beat the king (with margin), you take the throne

## Testnet

For testnet (netuid 460), the flow is identical but uses `--network test`:

```bash
python miner/commit.py \
  --wallet-name <your-wallet> \
  --wallet-hotkey <your-hotkey> \
  --image "docker.io/myuser/cacheon-miner:v1" \
  --digest "sha256:abc123..." \
  --network test \
  --netuid 460
```

## Troubleshooting

- **Image not found.** Make sure the image is publicly pullable without authentication. Test with `docker pull <image>@<digest>` from a clean machine.
- **Digest mismatch.** The on-chain digest must match the registry's manifest digest exactly. Use `docker inspect` to verify.
- **Already committed.** If you already committed on this hotkey, you cannot change it. Register a new hotkey.
