## Overview

Registration is three steps:

1. Push your Docker image to a public registry
2. Run `miner/commit.py` (pays the submission fee and commits in one flow)
3. The validator verifies payment on-chain and schedules evaluation

The validator scans for new commitments approximately every 5 minutes. When it finds yours, it pulls your image and evaluates it.

## Prerequisites

- [Install the Bittensor SDK](https://docs.learnbittensor.org/getting-started/installation). This is required in the environment used to run the [Cacheon miner commit script](#step-2-commit-on-chain).
- A Bittensor wallet registered on Bittensor mainnet subnet 14. For more information, see [Wallets in Bittensor](https://docs.bittensor.com/getting-started/wallets) and [Miner registration](https://docs.learnbittensor.org/miners#miner-registration).
- Enough TAO on your **coldkey** to cover the submission fee (default `0.1` TAO on mainnet when validators enable the gate).

## Step 1: Push your image

Tag and push your image to a public registry:

```bash
docker tag my-server:latest docker.io/myuser/cacheon-miner:v1
docker push docker.io/myuser/cacheon-miner:v1
```

Note the image digest (the `sha256:...` printed after push). You will need this.

To get the digest after pushing:

```bash
docker inspect --format='{{index .RepoDigests 0}}' docker.io/myuser/cacheon-miner:v1
```

## Step 2: Commit on-chain (payment + reveal)

Use the Cacheon commit script in the [Cacheon GitHub repository](https://github.com/latent-to/cacheon/blob/main/miner/commit.py). It runs **two steps in one command**:

1. **Pay the submission fee.** Your coldkey signs `Utility.batch_all([transfer, remark])`:
   - `Balances.transfer_keep_alive` sends the fee to the subnet treasury address
   - `System.remark` carries `cacheon:<hotkey>:<digest>` so the payment is tied to this image
2. **Reveal the commitment.** Your hotkey posts JSON with the image, digest, and payment proof.

```bash
python miner/commit.py \
  --wallet-name <your-wallet> \
  --wallet-hotkey <your-hotkey> \
  --image "docker.io/myuser/cacheon-miner:v1" \
  --digest "sha256:abc123..." \
  --fee 0.1 \
  --network finney \
  --netuid 14
```

On-chain commitment JSON (written by the script):

```json
{
  "image": "docker.io/myuser/cacheon-miner:v1",
  "digest": "sha256:abc123...",
  "payment_tx": "0xabc123...",
  "payment_block": "0xdef456..."
}
```

| Field           | Description                                           |
| --------------- | ----------------------------------------------------- |
| `image`         | Full Docker image reference (registry/repo:tag)       |
| `digest`        | Image manifest digest (`sha256:...`)                  |
| `payment_tx`    | Extrinsic hash of the fee batch (`0x` + 64 hex chars) |
| `payment_block` | Block hash where that extrinsic was included          |

Validators look up `payment_tx` with `substrate.retrieve_extrinsic_by_hash`, confirm the coldkey signer, an executed `Balances.Transfer` to the treasury, and the remark in the batch. One RPC lookup per submission; no separate payment database.

<Callout type="info" title="Resubmitting on the same hotkey">
  You can commit again on the same hotkey. Each new commitment needs a fresh fee payment and new
  `payment_tx` / `payment_block` fields. The validator uses the latest revealed commitment per
  hotkey; each commit block is tracked separately. Test locally before paying and committing.
</Callout>

<Callout type="warn" title="Register before you pay">
  The commit script checks that your hotkey is registered on the target netuid before submitting the
  fee. If payment succeeded but the commitment step failed, rerun with the printed `--payment-tx`
  and `--payment-block` flags to retry without paying twice.
</Callout>

## What happens after

The validator pulls your image and runs the [evaluation harness](/docs/evaluation/harness). If your score beats the current leader, you take the top spot.
