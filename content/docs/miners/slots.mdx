---
title: The slots you can target
---

A **slot** is a replaceable, narrowly-typed region of the model graph. The
validator owns the catalog; you may only target a slot that exists. You don't
invent slots — you pick one and write a kernel for its contract.

The full authoritative table is at [Slot catalog](/docs/reference/slots-table).
This page explains the *shape* of each kind so you can pick.

## The three kinds

| Kind | Boundary | What you get | Verification |
|---|---|---|---|
| **`op`** | a single fused op | `entry(x, out)` (inputs + the output to fill) | single-process, vs fp32 reference |
| **`block`** | a wider region, several ops fused behind one tensor boundary | `entry(q, k, v, out, ...)` or a `(prepare, forward)` pair | single-process vs fp32 reference |
| **`collective`** | spans GPUs | `entry(..., out, group)` — you're handed the process group | **distributed**: spawns world_size ranks, vs fp32 cross-rank reduce |

All three share the same cheat-resistant contract from
[The slot contract](/docs/architecture/slot-contract): you only ever fill an
output tensor the validator allocated, strictly upstream of the sampler.

## The slots, at a glance

Ten slots exist today. Grouped by what part of the model they're in:

**Activations / norms (ops — the simplest entry point):**

- `activation.silu_and_mul` — the MLP activation `silu(x[...,:d]) * x[...,d:]`.
  Contract: `entry(x, out)`. Correctness: `allclose`.
- `norm.rmsnorm` — RMSNorm (the pure normalization; the validator owns the
  residual-add fusion). Contract: `entry(x, weight, out, eps)`. Correctness:
  `allclose`.

**Attention (blocks):**

- `attention.sdpa` — full softmax-attention (GQA/MQA), prefill-scale. Contract:
  `entry(q, k, v, out, sm_scale, causal)`. Correctness: `matched_ratio` (0.99).
  Today this is mainly an offline/verify contract; the generic runtime seam
  dispatches decode and falls back for prefill unless a model-specific seam is
  installed.
- `attention.decode` — paged-decode attention. Contract:
  `entry(q, k, v, seq_lens, sm_scale, out)`. Correctness: `matched_ratio` (0.99).
  **Structurally eager** today (a per-step host-sync) — see
  [Graph-safe kernels](/docs/miners/graph-safety).
- `attention.msa_block_score` — the MiniMax-M3 sparse-attention indexer (which
  blocks to attend to). Contract: `entry(q, index_k, seq_lens, block_size, out)`.
  Correctness: `topk_overlap` — the output is a *selection*, not values. This is
  a live slot contract but not a generic seam; it is scoreable only in an arena
  that installs the matching MSA chokepoint.

**Mixture-of-experts (blocks):**

- `moe.fused_experts` — the expert MLPs (SwiGLU-style). A `(prepare, forward)`
  pair: `prepare(w13, w2)` runs once on raw weights; `forward(x, topk_ids,
  topk_weights, prepared, out)` per step. Correctness: `matched_ratio` (0.97).
  The validator replays the TP all-reduce after.
- `moe.fused_experts_reduce` — **the MoE block that owns its trailing TP
  all-reduce.** Same `(prepare, forward)` pair but `forward(...)` gets the
  `group` and fills `out` with the *already-reduced* result. This is the slot
  that can express the compute-comm **overlap** optimization (hide the reduce behind the
  expert GEMM). Correctness: `matched_ratio` (0.97).

**Cross-GPU collectives:**

- `collective.all_reduce` — the TP all-reduce. Contract: `entry(x, out, group)`.
  Correctness: `matched_ratio` (0.99).
- `collective.ar_residual_rmsnorm` — the fused AR + residual-add + RMSNorm
  epilogue (behind sglang's `--enable-flashinfer-allreduce-fusion`). Contract:
  `entry(x, residual, weight, eps, out_norm, out_residual, group)`. This was the
  first submitted kernel to surpass the pinned sglang baseline through Optima's
  scorer (2026-07-07).
- `collective.moe_finalize_ar_rmsnorm` — the **DEEP** fused epilogue: MoE
  finalize + AR + add + RMSNorm as ONE kernel. Enabled by a bundle-declared
  `dep_patches` diff against pinned flashinfer. See
  [Dependency patches](/docs/miners/dep-patches).

## How to pick

- **New to this?** Start with `activation.silu_and_mul` or `norm.rmsnorm`. They're
  the simplest contracts (`entry(x, out)`), they're op-correctness-gated by
  `allclose`, and they're graph-captured directly (no `graph_safe` declaration
  needed). The [20-minute walkthrough](/docs/miners/your-first-kernel) uses silu.
- **Want the big lever?** Decode at TP/EP scale is **comms-bound** (the
  all-reduce is ~32–43% of GPU time). The collective slots
  (`collective.ar_residual_rmsnorm`, `moe.fused_experts_reduce`,
  `collective.moe_finalize_ar_rmsnorm`) capture compute-comm overlap and fused
  epilogue opportunities. But they're distributed-verified
  and need a multi-GPU box to test.
- **Writing a quantized MoE?** `moe.fused_experts` with a `(prepare, forward)`
  pair — `prepare` does the weight repack/quant once, `forward` is the fused
  GEMM. For NVFP4 specifically, there's an
  [override-point](/docs/miners/override-points) path that lets you ship just an epilogue
  hook into a validator-owned base megakernel.

## Per-model specialization

Some slots are retargeted per model. The big example: MiniMax-M3's MoE uses the
**swigluoai** activation (`swish` with alpha 1.702), not plain silu, and its
NVFP4 experts are gated on **cosine** (not element-wise) correctness because
4-bit kernels carry ~6–12% per-element quant error. These are **validator-owned
facts** — you just name the model, and the slot is retargeted via the
`MODEL_PROFILES` registry (`optima/slots.py`). See
[Slot catalog](/docs/reference/slots-table) for the per-model details.

## The contract each slot shares

Whichever slot you pick, the contract is the same shape:

1. **You fill `out`.** The validator allocates it (shape/dtype/device/stride).
   You write into it (via `out.copy_(...)` or in-place writes). You return
   `None`. See [The kernel ABI](/docs/miners/kernel-abi).
2. **You stay upstream of the sampler.** Your output feeds the residual stream →
   downstream layers → sampler, all stock. You never produce tokens.
3. **The cheap verify gate uses high-precision ground truth**, not stock
   bit-patterns. The production model-level fidelity gate then compares
   stock/candidate behavior by KL or audit, depending on the arena.
4. **The validator measures everything itself.** You don't report your own
   throughput or correctness.

If your kernel needs to break any of these, it's not a core slot — it goes
through the fenced framework tier (the `setup()` hook + isolation). That's an
advanced, experimental path; see [the override points page](/docs/miners/override-points)
for the sanctioned composition pattern instead.
